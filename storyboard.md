# Pickleball Scoring Video Storyboard

This document describes the flow, scenes, scripts, and visual concepts of the Pickleball scoring animated video.

## Scene 1: Introduction to Pickleball Doubles Scoring
* **Visuals**: A bold title "How to Keep Score in Pickleball (Doubles)" in beautiful typography, sliding in or scaling up on a clean dark background.
* **Concept**: Grab the user's attention and state the objective clearly: understanding the three numbers announced before every serve.

## Scene 2: The Three Numbers Rule
* **Visuals**: Three big placeholders or text labels appear:
  - **First Number**: Serving Team's Score
  - **Second Number**: Receiving Team's Score
  - **Third Number**: Server Number (1 or 2)
* **Concept**: Clearly define what each number in "0 - 0 - 2" or "4 - 3 - 1" means. Use color coding for visual association.

## Scene 3: The Starting Exception (0 - 0 - 2)
* **Visuals**: A standard pickleball court is drawn (colored court, kitchen line, baseline, sidelines, and center line).
  - Serving Team (Team A: Red) on one side, Receiving Team (Team B: Blue) on the other.
  - Show players: Team A (Server 2 in the right court, teammate in the left court).
  - Highlight the Server with a glowing circle or an icon (e.g. racquet or ball indicator).
* **Concept**: Explain the "First Server Rule" exception. The game starts with the score `0 - 0 - 2` (or `0 - 0 - Second Server`). Explain that at the very beginning of the game, only one server gets to serve before a side-out. Hence, we start as server number 2.

## Scene 4: Score Progression (Serving Team Wins Point)
* **Visuals**: Serve and rally representation. Then, display a "Point!" text overlay.
  - The serving team shifts positions (Server switches sides: Right court to Left court, teammate moves to the Right court).
  - The score updates from `0 - 0 - 2` to `1 - 0 - 2`.
* **Concept**: Demonstrate that when the serving team wins a point, they score, and the servers switch sides (right to left or vice versa).

## Scene 5: Side-Out & Server Transition (Receiving Team Wins Rally)
* **Visuals**: Display a "Fault!" or "Side-out!" text overlay.
  - If they were on `1 - 0 - 2` and lose the rally:
    - Since it was Server 2, a **Side-out** occurs (serve moves to the other team).
    - Team B (Blue) now becomes the serving team. Their first server starts from the right court.
    - The new score is called: `0 - 1 - 1` (Team B score is 0, Team A score is 1, Server 1).
* **Concept**: Demonstrate how serving switches to the other team (side-out) and how the new server calls the score, starting with their own score first.
